# FPF Dialog Sessions

> Диалоговый режим на базе системного промпта `../../inbox/ПРОМПТ-Системный-FPF.md`.  
> Сохраняет ход мысли пилота между сессиями и накапливает паттерны мышления.

## Как работает — полный цикл

### Сценарий: сегодня

**Пилот:** «начинаем диалог»

**Kimi:**
1. Проверяет `memory/conversations/` — есть ли диалоги со статусом `active`?
2. Если `active` нет → спрашивает: «Начать новый или продолжить один из закрытых?»
3. Если пилот выбирает **новый** → создаёт файл `memory/conversations/FPF-YYYY-MM-DD-{тема}.md`, применяет промпт, начинает
4. Если пилот выбирает **старый** → показывает список закрытых диалогов (дата + тема + последний turn), меняет статус на `active`, читает файл, продолжает с места остановки

**В процессе:**
- Каждый turn записывается в файл сразу после ответа Kimi
- Пилот может сказать «сохрани диалог» — коммит без закрытия

**Пилот:** «закрывай диалог»

**Kimi:**
1. Меняет статус на `closed`
2. Дописывает секцию **Capture**:
   - Паттерны мышления (какие различения использует пилот, где смешивает)
   - Ключевые выводы
   - Открытые вопросы
3. Коммитит файл

---

### Сценарий: завтра

**Пилот:** «начинаем диалог»

**Kimi проверяет `active`:**
- Вчерашний диалог закрыт → `active` нет
- **Спрашивает:** «Начать новый или продолжить вчерашний?»

**Если пилот говорит «продолжим вчерашний»:**
- Kimi находит файл, меняет статус на `active`
- Читает Capture → видит, на чём остановились и что осталось открытым
- Говорит: «В прошлый раз мы остановились на X. Остался открытый вопрос Y. Продолжим оттуда или сначала?»

**Если пилот говорит «новый про [тема]»:**
- Создаёт новый файл, старый остаётся закрытым

---

### Что происходит при продолжении закрытого диалога

Kimi не просто читает последний turn. Он читает **весь файл**, включая Capture. Это значит:
- Он помнит, какие различения уже были сделаны
- Видит, какие вопросы остались открытыми
- Знает паттерны мышления пилота из прошлого диалога
- Не начинает с нуля, а продолжает с накопленным пониманием

## Пример триггеров

| Намерение | Фраза пилота | Что делает Kimi |
|-----------|-------------|-----------------|
| **Начать** (с проверкой) | «начинаем диалог» | Проверяет active → спрашивает новый/старый |
| **Начать новый** (сразу) | «начинаем новый диалог про [тема]» | Создаёт файл, старые не трогает |
| **Продолжить конкретный** | «продолжим диалог про [тема]» | Ищет по теме, статус → active |
| **Сохранить** | «сохрани диалог» | Коммит файла |
| **Закрыть** | «закрывай диалог» | Capture + статус closed + коммит |
| **Зафиксировать выводы** | «зафиксируй выводы» | Дописывает Capture в active-диалог |
