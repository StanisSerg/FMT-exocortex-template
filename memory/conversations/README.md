# FPF Dialog Sessions

> Диалоговый режим на базе системного промпта `../../inbox/ПРОМПТ-Системный-FPF.md`.  
> Сохраняет ход мысли пилота между сессиями и накапливает паттерны мышления.

## Как работает

### 1. Начало диалога
Пилот говорит:

> «Начнём FPF-диалог по теме X»

Kimi:
1. Создаёт файл `memory/conversations/FPF-YYYY-MM-DD-{тема}.md` по шаблону
2. Применяет системный промпт из `inbox/ПРОМПТ-Системный-FPF.md`
3. Ведёт диалог в естественном тоне (без объявления методов)
4. **После каждого обмена репликами записывает turn в файл**

### 2. В процессе диалога
- Файл дописывается turn за turn'ом
- Если контекст в чате сбился (новая сессия, compaction) — Kimi читает файл и продолжает
- Пилот может сказать «сохрани диалог» — Kimi делает git commit файла

### 3. Продолжение диалога
Пилот говорит:

> «Продолжим диалог FPF-YYYY-MM-DD-{тема}»

Kimi читает файл с последнего turn'а и продолжает.

### 4. Закрытие диалога
Пилот говорит:

> «Закроем диалог»

Kimi:
1. Меняет статус на `closed`
2. Дописывает секцию **Capture** — паттерны мышления пилота, ключевые различения, решения
3. Коммитит файл
4. При необходимости — переносит критичные insight'ы в `memory/` или `pack/`

## Что даёт

- **Сохранение контекста:** диалог не растворяется в истории чата
- **Накопление модели мышления:** capture'ы показывают, как пилот мыслит — какие критерии использует, где смешивает, какие решения принимает
- **Повторяемость:** можно вернуться к диалогу через месяц и продолжить с того же места
- **Связь с РП:** диалог можно привязать к WP (например, WP-99 FPF-диагностика) через frontmatter

## Пример триггеров

| Намерение | Фраза пилота |
|-----------|-------------|
| Начать | «Начнём FPF-диалог про [тема]» |
| Продолжить | «Продолжим диалог [тема/дата]» или «Вернёмся к диалогу про [тема]» |
| Сохранить | «Сохрани диалог» |
| Закрыть | «Закроем диалог» / «Хватит на сегодня» |
| Capture | «Зафиксируй выводы» (внутри диалога или при закрытии) |
