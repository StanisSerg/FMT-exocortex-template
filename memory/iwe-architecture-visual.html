WP Gate — блокеры

  • Нет WeekPlan → СТОП
  • Бюджет недели перегружен → Предупреждение
  • Open-ended WP → Запрещены

Quick Close — что фиксируется

  • Коммиты за сессию
  • Статус WP (partial / done)
  • Уроки (≤8 в MEMORY.md)
  • Следующий шаг (next action)

5. Роли (Agents) и их функции

ID Роль FX Исполнитель Типичные задачи
R1 Стратег — Скилл Day Open/Close, Week Close, стратсессии, wp-new, run-protocol
R2 Экстрактор — Скилл KE в Pack, inbox check, ontology sync, apply-captures
R5 Архитектор FX5 Скилл + inline ArchGate (7 критериев ЭМОГССБ), ADR, FPF routing
R6 Кодировщик FX5, FX8 Inline + скилл Код, рефакторинг, iwe-update, audit-installation
R23 Верификатор — Sub-agent Проверка артефактов по эталону. Context isolation
R24 Аудитор — Sub-agent + скилл audit-installation: 6 компонентов, verdict ✅/⚠️/❌
R8 Синхронизатор — Скрипт + скилл update.sh, template-sync, drift detection, backup
R9 Шаблонизатор FX8 Скилл iwe-update, extend, pack-new
Ключевой принцип: Context isolation (VR.SOTA.002) — Аудитор и Верификатор не получают историю сессии. Оценивают ТОЛЬКО по отчёту.

dry-run-gate.sh

Блокирует write-tools при наличии sentinel-файла. Используется в audit-installation для smoke-test без побочных эффектов.

TTL: 10 мин (kill -9 защита)

wp-gate-reminder.sh

Проверяет наличие WP в MEMORY.md перед началом работы. Нет WP = СТОП.

wakatime-heartbeat.sh

Отправляет heartbeat в WakaTime каждые 2 минуты при работе в Kimi CLI.

10. Ключевые принципы (чеклист понимания)

1. L1 = Immutable

FMT-exocortex-template никогда не правится руками. Только через update.sh. Все кастомизации — в L3 (extensions/, params.yaml).

2. WP Gate = Блокер

Нет WP в MEMORY.md → работа не начинается. Нет бюджета в WeekPlan → предупреждение.

3. SYNTH ≠ Rules

SYNTH — только для обучения. Правила пишутся от индивидуальных SoTA. ArchGate обязателен.

4. Context Isolation

Аудитор и Верификатор — sub-agents без истории сессии. Оценивают только артефакт.

5. Atomic Writes

wp-new = запись в 5 мест атомарно. Пропуск места = невалидный WP.

6. Dry-Run Contract

Sentinel-файл + dry-run-gate.sh = безопасное тестирование без write-побочных эффектов.
